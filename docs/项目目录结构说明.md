# 企业知识库智能助手 —— 项目目录结构说明 V1

## 总体结构

```
knowledge-base-assistant/
├── frontend/              # Vue 3 前端项目
├── backend-java/          # Spring Boot 业务服务
├── backend-python/        # FastAPI AI 服务
├── sql/                   # 数据库建表脚本
├── docker/                # Docker Compose 编排配置
├── docs/                  # 需求、设计、计划文档
└── data/files/            # 上传文件存储目录（共享卷）
    └── {kb_id}/           # 按知识库 ID 分组
```

### 前端职责说明

| 模块 | 职责 |
|------|------|
| `api/` | 每个文件对应一个后端模块的接口调用，统一通过 `utils/request.js` 发送 HTTP 请求 |
| `views/` | 每个目录对应一个页面，页面内可引用 `components/` 中的通用组件 |
| `stores/` | 用 Pinia 管理全局状态：登录态（auth）和当前会话消息（chat） |
| `router/` | 配置路由表 + `beforeEach` 守卫：未登录跳转登录页，普通用户无法访问管理页 |
| `utils/request.js` | axios 实例，请求头自动注入 `Authorization: Bearer {token}`，响应拦截统一处理 401 |

### 后端分层职责

| 层级 | 职责 | 依赖 |
|------|------|------|
| `controller` | 接收 HTTP 请求，参数校验，调用 Service，返回 Result | 无业务逻辑 |
| `service` | 核心业务逻辑，事务管理，权限校验 | 依赖 Mapper 和其他 Service |
| `mapper` | 数据库操作（MyBatis-Plus BaseMapper） | 对应 Entity |
| `entity` | 数据库表映射，`@TableName` + `@TableId` | 与表字段一一对应 |
| `dto` | 请求体和响应体的数据结构 | 与 Entity 分离，避免接口暴露表结构 |
| `security` | JWT 拦截认证，Spring Security 集成 | — |
| `client` | 与 FastAPI 的内部 HTTP 通信 | — |
| `config` | Spring Boot 配置类 | — |
| `common` | 统一响应体、异常处理、工具类 | — |

### 包名规范

- 基础包：`com.example.kbassistant`
- Controller 统一路径前缀：`/api`
- 内部接口（调用 FastAPI）统一路径前缀：`/internal`

### FastAPI 模块职责

| 模块 | 职责 |
|------|------|
| `routers/` | 定义 HTTP 接口，参数校验，调用 services，返回响应 |
| `schemas/` | Pydantic 模型，定义请求体和响应体的数据结构 |
| `services/parser/` | 每种文件格式一个解析器，统一继承基类，输出标准化文本结构 |
| `services/chunker.py` | 按标题/段落切片，回退到固定长度切片，输出 chunk 列表 |
| `services/embedder.py` | 调用 SiliconFlow Embedding API，返回向量 |
| `services/retriever.py` | 查询 Qdrant，按 knowledge_base_id 过滤，返回 TopK |
| `services/qa_generator.py` | 拼接 Prompt + 检索结果，调用 DeepSeek API，返回答案 |
| `services/citation.py` | 将检索结果组装为前端可展示的引用信息 |

## 五、依赖关系总结

```
前端 (Vue 3)  ──HTTP──>  Spring Boot (8080)  ──HTTP──>  FastAPI (8000)
                              │                              │
                              ├── MySQL (3306)               ├── Qdrant (6333)
                              └── Redis (6379)               └── SiliconFlow API (外部)
```

- Spring Boot 是所有业务入口，前端只与 Spring Boot 通信
- FastAPI 不对外暴露，只接收 Spring Boot 的内部调用
- 文件通过共享卷 `./data/files/` 传递，不走 HTTP 二进制流
